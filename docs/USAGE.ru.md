## 🚀 Быстрый старт

### 🎮 Запуск игры с ленты (3 клика)

```
1. Нажмите 📂 Open (или F3) и выберите файл .tap или .tzx
   → Окно Devices откроется автоматически на вкладке Tape

2. Нажмите 🚀 Quick Load (кнопка-ракета)
   → Лента перемотается, отправится команда LOAD "", начнётся воспроизведение

3. Дождитесь окончания загрузки
   → Играйте! 🎉
```

### 🕹️ Игра с геймпадом

```
1. Подключите USB/Bluetooth геймпад
   → Он определится автоматически (отобразится в строке статуса)

2. Нажмите Tab или 🕹️ Joystick пока не выберется "Kempston"
   → Строка статуса покажет: "Virt. joystick 3/5: Kempston"

3. Используйте геймпад:
   - Стик/D-pad → направления джойстика
   - X → Space, Y → Enter
   - Остальные кнопки → огонь
```

### 💾 Сохранение прогресса

```
1. Нажмите F5 или 💾 Save
2. Выберите место и введите имя файла с расширением .z80 (например, `game.z80`)
3. Для загрузки позже: File → Open → выберите ваш .z80 файл
```

## 📂 Открытие файлов

**Поддерживаемые форматы:**

| Тип | Расширения | Примечания |
|------|-----------|------------|
| **ROM** | `.rom`, `.bin` | Макс. 16 КБ (меньше можно) |
| **Snapshot** | `.z80`, `.sna`, `.mgc` | Загрузка любых; сохранение только в `.z80` (v1.0) |
| **Лента** | `.tap`, `.tzx` | Полная поддержка блоков TZX |

**Как открыть:**

- 🔘 **Кнопка**: Нажмите 📂 Open (F3) → выберите файл
- 🖱️ **Drag & Drop**: Перетащите любой поддерживаемый файл на окно эмулятора *(рекомендуется при работе с множеством файлов)*

**Загрузка собственного ROM:**
- Используйте File → Open (F3) и выберите файл `.rom` или `.bin`
- Эмулятор перезагрузится с новым ROM
- Для возврата к оригинальному ROM нажмите кнопку **Restore ROM** на вкладке System

## 💾 Сохранение состояний

**Текущая версия (v1.0):**

- ✅ Сохранение только в формате `.z80`
- 📁 Расположение: выбирается пользователем через стандартный диалог
- 📝 **Расширение `.z80` нужно указывать вручную** — при сохранении введите имя файла с расширением, например `game.z80`
- ♻️ **Загружаемые**: Сохранённые `.z80` файлы можно открыть через Open (F3)

## 🎮 Устройства ввода

### Клавиатура

**Соответствие клавиш:**

| Клавиша Spectrum | Клавиша PC |
|-----------------|------------|
| Стрелки (5,6,7,8) | Стрелки |
| Кнопка огня | `Left Alt` |
| Caps Shift | `Right Shift` |
| Symbol Shift | `Right Ctrl` |
| Space | `Space` |
| Enter | `Enter` |

> 💡 Полная раскладка: см. виртуальную клавиатуру (Devices → Keyboard).

### Виртуальные джойстики

**5 режимов** (переключение: `Tab` или 🕹️):

| Режим | Описание | Для чего |
|------|----------|----------|
| **Cursor control** | Эмулирует **клавиши со стрелками в режиме BASIC**: перемещение курсора по экрану с удержанием Caps Shift | Навигация в BASIC, редактор программ |
| **Kempston** | Широко используемый сторонний интерфейс | Большинство игр |
| **Sinclair Interface 2** | Поддержка двух джойстиков (левый/правый порт) | Игры с двойным управлением |
| **Protek** | Также известен как AGF | Отдельные игры |
| **Cursor** | Эмулирует **непосредственное нажатие клавиш 5–8 и 0** (без модификаторов) | Игры, использующие эти клавиши как джойстик |

**Краткое пояснение:**
- В ZX Spectrum физические клавиши **5, 6, 7, 8** выполняют роль стрелок
- В BASIC они перемещают курсор **только при зажатом Caps Shift** (режим **Cursor control**)
- В играх они часто используются как джойстик напрямую (режим **Cursor**)

**Строка статуса**: Показывает текущий режим, например: `Virt. joystick 3/5: Kempston`

### Поддержка геймпадов

**Автоопределение**: Подключайте/отключайте в любой момент через USB или Bluetooth.

**Назначение кнопок** (фиксированное для простоты):

| Вход геймпада | Эмулируемое действие |
|--------------|---------------------|
| Левый/правый стик, D-pad | Направления джойстика |
| Кнопка `X` | `Space` |
| Кнопка `Y` | `Enter` |
| Все остальные кнопки (`A`, `B`, плечи, триггеры) | Кнопка огня |

**Выбор геймпада при подключении нескольких:**

Если подключено более одного геймпада, появится **модальное диалоговое окно** с предложением выбрать активный контроллер. Эмуляция будет приостановлена до выбора.

> ⚙️ **Примечание дизайна**: Назначение кнопок не настраивается для упрощения интерфейса. Значения по умолчанию выбраны для типичного использования.

### Поддержка мыши

**3 режима** (переключение: 🖱️; без горячей клавиши):

| Режим | Описание |
|------|----------|
| **Disconnected** *(по умолчанию)* | Эмуляция мыши отключена |
| **Kempston** | Стандартная эмуляция мыши Kempston |
| **Kempston (inverted)** | Кнопки поменяны местами (для совместимости с программами, использующими обратное назначение битов) |

**Как использовать**:

1. Выберите Kempston или Kempston (inverted)
2. **Кликните внутри окна эмулятора** → мышь захвачена
3. Двигайте/кликайте для взаимодействия с эмулируемым ПО
4. Нажмите `Esc` → освободить захват мыши

**Как отключить мышь:**
- Снова нажмите 🖱️ и выберите **Disconnected**
- Или просто выйдите из режима захвата нажатием `Esc`

> ⚠️ **Почему по умолчанию Disconnected?** Захват мыши не позволяет использовать её за пределами окна эмулятора. Поскольку мышь требуется редко, она отключена по умолчанию для удобства.

## 📼 Эмуляция ленты

### Автоматическая загрузка (APDS)

**APDS** (Automatic Polling Detection System) автоматизирует воспроизведение ленты:

```
✅ Spectrum опрашивает порт ленты → APDS запускает воспроизведение
✅ Загрузка завершена → APDS автоматически останавливает ленту
```

**Два режима** (переключатель на вкладке Tape):

| Режим | Поведение | Когда использовать |
|------|----------|-------------------|
| **Strict** *(по умолчанию)* | Срабатывает только если **не опрашивается строка клавиатуры** (кроме строки с пробелом) | Большинство игр и загрузчиков |
| **Lenient** | Позволяет детектировать ленту даже если опрос клавиатуры похож | Программы с нестандартной обработкой ввода |

**Рекомендуемый порядок действий**:

```
1️⃣ Попробуйте загрузку с APDS Strict (по умолчанию)
   → Работает для большинства ПО

2️⃣ Если загрузка зависает неожиданно (без подсказок на экране)
   → Переключитесь на APDS Lenient, попробуйте сначала

3️⃣ Если всё равно не работает
   → Отключите APDS, используйте ручное управление (см. ниже)
```

**Известные ограничения APDS** (требуют ручного управления):

| Проблема | Почему происходит | Решение |
|----------|------------------|---------|
| Очень короткие блоки ленты | APDS не может среагировать быстрее ~0.6 сек виртуального времени | Отключить APDS, управлять вручную |
| Загрузчики пропускают блоки | APDS останавливает ленту → загрузчик ищет → APDS перезапускает тот же блок → цикл | Отключить APDS |
| Защита копирует паузы опроса | APDS интерпретирует паузу как "готово" | Отключить APDS |

> 👁️ **Совет по отладке**: Красная стрелка в списке блоков показывает текущую позицию воспроизведения — помогает определить, где загрузка зависла.

### Ручное управление лентой

**Управление** (вкладка Devices → Tape):

| Кнопка | Действие |
|--------|----------|
| ▶️ Play | Начать воспроизведение с текущего блока |
| ⏹️ Stop | Остановить воспроизведение (курсор на месте) |
| ⏮️ Rewind | Переместить курсор на 1 блок назад |
| ⏭️ Fast Forward | Переместить курсор на 1 блок вперёд |
| ⏪ Rewind to Start | Перейти к первому блоку + сбросить внутреннее состояние (стек циклов/вызовов) |
| 🗑️ Clear | Выгрузить образ ленты ("извлечь кассету") |

**Список блоков** (только для информации):

| Колонка | Показывает |
|---------|------------|
| Type | Тип блока (Std. speed, Pure data и т.д.) |
| Size | Размер блока (hex) |
| Description | Технические детали (pilot, pause и т.д.) |

> 🔒 Список **только для чтения** для предотвращения случайного изменения порядка блоков.

### Быстрая загрузка

**🚀 Кнопка-ракета** = загрузка одним кликом:

```
1. Перематывает на начало (Rewind to Start)
2. Отправляет команду LOAD на Spectrum
3. Запускает воспроизведение ленты
```

**Тип команды** (выпадающий список рядом с ракетой):

| Опция | Команда BASIC | Для чего |
|--------|--------------|----------|
| **Program** *(по умолчанию)* | `LOAD ""` | Загрузка программ |
| **Code** | `LOAD "" CODE` | Загрузка машинного кода |
| **Screen** | `LOAD "" SCREEN$` | Загрузка изображений экрана |

**Рабочий процесс**:
```
Открыть файл ленты → Нажать 🚀 → Готово!
```

## ⚙️ Настройки системы

**Ползунок Performance**:

- 🐢 **x1** *(по умолчанию)*: Эмуляция в реальном времени (1:1)
- 🚀 **x2…x10**: Ускорение эмуляции

**Hooks** (автоматическое изменение скорости):

| Hook | По умолчанию | Назначение |
|------|-------------|------------|
| On tape start | x10 | Быстрая загрузка |
| On tape stop | x1 | Возврат к нормальной скорости для игры |

- 🔢 Значения настраиваются (1–10)
- ☑️ Можно полностью отключить

**Кнопки действий**:

| Кнопка | Действие |
|--------|----------|
| **NMI** | Сгенерировать Non-Maskable Interrupt (аппаратная кнопка на реальном Spectrum) |
| **Restore ROM** | Вернуть оригинальный спектрум.ром + сброс эмулятора *(без подтверждения)* |

> ⚠️ **Restore ROM** заменяет любой custom ROM немедленно. Полезно, если плохой дамп ROM вызывает нестабильность.

## ⌨️ Виртуальная клавиатура

**Расположение**: Откройте окно Devices (кнопка в панели инструментов) → перейдите на вкладку **Keyboard**.

![Virtual Keyboard](docs/images/screenshot-keyboard.png)

**Управление мышью**:

| Действие | Результат |
|----------|-----------|
| **ЛКМ** по клавише | Нажать эту клавишу один раз в эмуляторе |
| **ПКМ** по Caps Shift / Symbol Shift | **Зафиксировать** модификатор до следующего нажатия |

**Пример: ввод красного ключевого слова**
```
1. ПКМ по "Caps Shift" → он остаётся нажатым
2. ЛКМ по "A" → отправляется команда EDIT (Caps+A)
3. Caps Shift автоматически отпускается
```

> ✅ Позволяет вводить сложные комбинации только мышью — не нужно удерживать две кнопки одновременно.

## 📊 Строка статуса

Показывает текущее состояние эмулятора в реальном времени:

| Элемент | Пример | Значение |
|---------|--------|----------|
| Virt. joystick | `3/5: Kempston` | Текущий режим джойстика (3 из 5) |
| Gamepad | `SHANWAN Android Gamepad` | Имя подключённого геймпада |
| Mouse | `1/3: Disconnected` | Текущий режим мыши |
| Perf | `x1` или `x10` | Множитель скорости эмуляции |

> 🔄 Обновляется автоматически при изменении настроек.
